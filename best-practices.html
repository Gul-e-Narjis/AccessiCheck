Best Practices

Automated scans only catch ~30-40% of real accessibility problems. These six habits cover what a scanner can't — review them once per project.

  • Body text should hit at least 4.5:1 contrast against its background (3:1 for large text, 18pt+).
  • Never use color alone to show meaning — pair red/green status with an icon or label too.
  • Check your links: they need to be distinguishable from body text by more than color alone (underline helps).
  • Test your palette in grayscale — if two elements become indistinguishable, contrast is too low.
  • Every clickable thing should also work with Tab + Enter/Space — no mouse-only interactions.
  • Keep a visible focus ring on every focusable element. Never set outline: none without a replacement.
  • Tab order should follow the visual reading order, not the order elements happen to appear in the DOM/CSS.
  • Avoid keyboard traps — a user should always be able to Tab or Escape their way out of a menu or modal.
  • Write alt text that describes purpose, not just appearance — "Submit order" beats "blue button".
  • Keep one heading hierarchy per page (h1 → h2 → h3) — don't skip levels just to get a font size.
  • Use ARIA only when native HTML can't do the job — a real <button> is better than a styled <div role="button">.
  • Mark purely decorative images with alt="" so screen readers skip them instead of reading a filename.
  • Every input needs a real, associated <label> — placeholder text is not a substitute for a label.
  • Mark required fields with text, not just an asterisk or color (e.g. "Email (required)").
  • Error messages should say what's wrong and how to fix it, placed right next to the field.
  • Group related fields (like a billing address) with <fieldset> and <legend>.
  • Provide captions for video and a transcript for audio-only content.
  • Never autoplay video/audio with sound — let the user start it themselves.
  • Respect prefers-reduced-motion — pause non-essential animation for users who request it.
  • Avoid content that flashes more than 3 times per second (seizure risk).
  • Content should reflow cleanly at 200% browser zoom without horizontal scrolling.
  • Touch targets (buttons, links) should be at least 44×44px so they're easy to tap accurately.
  • Don't lock orientation — let the page work in both portrait and landscape.
  • Test with text-size-only zoom too (not just pinch-zoom) — some users only increase font size.
